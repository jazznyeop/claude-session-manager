# Session Manager

저장해둔 Claude Code / Codex 세션을 이름으로 관리하고, 새 터미널 창에서 이어서 열어주는 스킬.

모든 작업은 번들 스크립트로 처리한다 (직접 JSON 파일을 편집하지 말 것 — 스크립트가 형식과 작업폴더 자동 감지를 보장한다).

## 스크립트 경로 (매 명령 공통 — 먼저 읽을 것)

이 스킬은 **단독 설치**(`~/.claude/skills/`에 복사)와 **플러그인 설치**(`/plugin install`) 둘 다 지원한다. 스크립트 위치가 설치 방식에 따라 다르므로, **매 명령은 경로를 자동 결정해서 한 번의 Bash 호출로 실행할 것** (Bash 호출 사이엔 셸 변수가 유지되지 않으니 반드시 한 줄에 이어서):

```bash
S="$HOME/.claude/skills/session-manager/scripts/sessions.py"; [ -f "$S" ] || S="$CLAUDE_PLUGIN_ROOT/skills/session-manager/scripts/sessions.py"; python3 "$S" <서브커맨드>
```

아래 예시들은 편의상 `python3 "$S" <cmd>`로 줄여 쓴다. 실제 실행 시 위의 `S=...; [ -f "$S" ] || S=...;` 결정 부분을 **같은 명령 앞에 붙여** 한 번에 실행할 것.

북마크 파일: `~/.claude/session-bookmarks.json` (설치 방식과 무관하게 항상 사용자 홈)

## 사용자 요청 → 동작 매핑

### "지금 이 세션 저장해줘" (이름 함께 또는 없이)
이름이 없으면 지금 대화 주제로 짧은 이름을 제안하고 확인받은 뒤:
```bash
python3 "$S" current "이름"
```
현재 세션 ID(`CLAUDE_CODE_SESSION_ID`)와 작업폴더가 자동 저장된다. **저장 시 대화기록(`.jsonl`)이 `~/.claude/session-archive/`로 자동 복사(아카이브)되어, Claude Code가 30일 뒤 원본을 자동삭제해도 복원해서 이어갈 수 있다.** 저장 출력에 `📦 아카이브됨`이 뜨면 영구보관된 것. `⚠️ 아카이브 못함`이 뜨면 대화기록을 못 찾은 것이니 세션 ID를 확인할 것.

### "이 세션 저장해줘" + resume 명령어 붙여넣기
붙여넣은 명령에서 도구(claude/codex)와 UUID를 추출해서:
```bash
python3 "$S" add "이름" claude|codex UUID
```
작업폴더는 세션 기록에서 자동으로 찾아낸다. 못 찾으면 홈 폴더로 저장되니, 사용자가 특정 프로젝트 폴더 작업이었다고 하면 `--dir /경로` 추가.

### "세션 목록 보여줘"
```bash
python3 "$S" list
```
상태 표시: `OK`=원본 살아있음, `OK(아카이브됨)`=원본+아카이브 둘 다, `ARCHIVED(보관됨, 복원가능)`=원본은 삭제됐지만 아카이브로 복원해 이어갈 수 있음, `DEAD(기록 삭제됨)`=원본도 아카이브도 없어 복구 불가. 사용자에게 표로 보기 좋게 정리해서 보여주고, DEAD가 있으면 알려줄 것.

### "OO 세션 열어줘 / 불러와줘 / 이어서 하자"
```bash
python3 "$S" open "OO"
```
새 Terminal 창이 열리고 해당 세션이 이어진다. 이름은 일부만 맞아도 된다(부분 일치). 여러 개가 매칭되면 스크립트가 후보를 알려주니 사용자에게 어느 것인지 물어볼 것.

자동 실행 말고 명령어만 원하면:
```bash
python3 "$S" cmd "OO"
```

### "OO 세션 코드 알려줘 / 실행코드 / 코드 줘 / 클립보드에 넣어줘 / 복사해줘" (기본 동작 = clip)
```bash
python3 "$S" clip "OO"
```
resume 명령(경로 없이 `claude/codex resume UUID` 형태 — bare)을 macOS 클립보드(`pbcopy`)에 복사하고, 어떤 세션·도구·명령을 넣었는지 출력한다. 사용자는 원하는 터미널에서 Cmd+V로 붙여넣고 실행하면 된다. `open`과 달리 새 창을 열지 않으므로, 사용자가 이미 띄워둔(대개 그 프로젝트 폴더의) 터미널에서 직접 이어가고 싶을 때 쓴다 — 그래서 clip/cmd은 `cd`를 붙이지 않는다(`open`만 새 창이라 `cd`가 붙는다). 출력에 표시된 세션 이름을 사용자에게 그대로 알려줄 것.

**기본 규칙 — "코드 달라"는 요청 처리:**
- 사용자가 저장된 세션 이름(부분이라도)으로 "코드 줘 / 세션 코드 알려줘 / 실행코드"라고 하면, **되묻지 말고 바로 `clip "OO"`를 실행**하고 복사된 명령어를 그대로 출력할 것. 그게 곧 답이다.
- **모호할 때만 물어볼 것**: 부분일치가 여러 개거나(스크립트가 후보 목록 반환), 사용자가 댄 이름이 어느 세션인지 확실치 않으면 — 임의로 고르지 말고 `list`로 후보를 표로 보여주고 "어느 거?"라고 물어볼 것.
- 매칭 0개면 `list` 전체를 보여주고 사용자에게 확인.
- DEAD(세션 기록 삭제됨) 항목이면 clip이 거부하니, 그 사실을 알리고 북마크 삭제를 제안할 것.

### "OO 영구보관 / 아카이브해줘" (이미 저장된 세션의 기록을 지금 보관)
```bash
python3 "$S" archive "OO"
```
`add`/`current`로 저장할 때 자동 아카이브되지만, 예전에 저장만 해두고 아직 살아있는 세션을 지금 명시적으로 보관하고 싶을 때 쓴다. 대화기록이 이미 삭제됐으면(디스크에 없음) 아카이브 불가 — 그 사실을 알릴 것. `open`/`clip`/`cmd`는 아카이브된 세션을 열 때 원본을 자동으로 복원한 뒤 resume 명령을 낸다(복원 시 `📦 복원함` 표시).

### "OO를 할 일로 추가해줘" (아직 세션이 없는 작업)
```bash
python3 "$S" todo "이름"
```
세션 ID 없이 이름만 목록에 올라가고, list에서 `TODO(세션 없음)`으로 표시된다. 나중에 그 작업을 시작한 세션을 **같은 이름으로** `add`나 `current`로 저장하면 TODO 항목에 자동으로 연결된다 (덮어쓰기 에러 없음 — 단, 이미 ID가 있는 항목은 보호됨).

### "OO 세션 삭제해줘" / "이름 바꿔줘"
```bash
python3 "$S" remove "OO"
python3 "$S" rename "옛이름" "새이름"
```

### "OO를 XX 카테고리로 분류해줘"
```bash
python3 "$S" cat "OO" "카테고리"
python3 "$S" cat "OO" ""   # 분류 해제
```
`list`가 카테고리별로 묶어서 출력한다. 카테고리 이름은 사용자가 자유롭게 정한다(예: `work` / `personal`). 스크립트의 `CATEGORY_ORDER`에 적힌 이름이 목록 위쪽에 먼저 나오고, 나머지는 그 뒤, 분류 없는 항목은 맨 아래 `미분류`로 모인다. 새 세션을 저장(`current`/`add`/`todo`)할 때 주제가 기존 카테고리에 맞으면 저장 직후 `cat`으로 분류까지 해줄 것.

## 주의사항

- 세션 기록은 `cleanupPeriodDays` 설정 기간이 지나면 Claude Code가 자동 삭제한다. **단, 저장 시 자동 아카이브되므로 그 세션은 `ARCHIVED`로 남아 복원 가능하다.** 아카이브 전에 삭제됐거나 아카이브에 실패한 세션만 `DEAD`가 되며, 이건 복구 불가 — 사용자에게 설명하고 북마크 삭제를 제안할 것.
- 아카이브 사본은 `~/.claude/session-archive/<tool>-<id>.jsonl`에 쌓인다. 세션당 1개이며 큰 대화는 용량이 클 수 있다. `remove`로 북마크를 지우면 아카이브 사본도 함께 삭제된다.
- codex 세션은 현재 세션 자동 감지가 안 된다. 사용자가 codex 세션을 저장하려면 resume 명령어(UUID 포함)를 직접 붙여넣어야 한다.
- `open`은 macOS Terminal.app 새 창을 연다. 같은 세션을 두 창에서 동시에 열면 충돌할 수 있으니, 이미 열려 있을 법하면 사용자에게 확인.
- `pbcopy`(clip)·`osascript`+Terminal.app(open)은 macOS 전용이다. 리눅스/윈도우에선 `list`/`add`/`cmd`/`remove`/`rename`/`cat`/`todo`만 동작한다.
